Overview

Base URL: https://transcript.fluximetry.com/api/v1

All responses are JSON. Errors return {"error": "message"} with appropriate HTTP status codes.

Rate limits: 5 requests/second, 30 uploads/minute per IP. Health endpoint is exempt.

Jobs

POST /api/v1/jobs

Upload file(s) and start transcription. For files over 50MB, use chunked upload instead.

FieldTypeRequiredDescription
filesfileYesOne or more audio/video files (multipart)
modelstringNotiny, base, small, medium, large (default: base)
languagestringNoLanguage code or "auto" (default: auto)
animated_quotesstringNo"true" to enable voice inflection analysis
two_list_quotesstringNo"true" to enable two-list quote detection
speaker_diarizationstringNo"true" to enable speaker detection
num_speakersintegerNoExpected speakers, 1-20 (auto-detect if omitted)
speaker_namesstringNoComma-separated names (e.g., "Alice,Bob")
{
  "success": true,
  "jobs": [{"job_id": "uuid", "filename": "meeting.mp4"}]
}

GET /api/v1/jobs

List all jobs. Filter with ?status=queued|processing|completed|failed

{
  "jobs": [{"id": "uuid", "status": "completed", "filename": "...", ...}]
}

GET /api/v1/jobs/{id}

Get details for a specific job including output files.

{
  "id": "uuid", "filename": "meeting.mp4", "status": "completed",
  "progress": 100, "model": "base", "language": "en",
  "output_files": [{"name": "meeting_transcription.txt", "size": 4096}],
  "created_at": "...", "completed_at": "..."
}

DELETE /api/v1/jobs/{id}

Cancel a queued/processing job or delete a completed/failed job. Kills subprocess if running. Cleans up all files.

{"success": true, "message": "Job uuid deleted"}

GET /api/v1/jobs/{id}/files/{filename}

Download an output file from a completed job.

Returns the file content with Content-Disposition: attachment.

Chunked Upload

For files over 50MB (required behind Cloudflare Tunnels with 100MB limit).

POST /api/v1/uploads

Initiate a chunked upload session. Send JSON body.

FieldTypeRequiredDescription
filenamestringYesOriginal filename
total_chunksintegerYesTotal number of chunks (1-200)
Plus all transcription options (model, language, etc.)
{"upload_id": "uuid", "filename": "large_file.mp4", "total_chunks": 7, "chunk_size_mb": 50}

POST /api/v1/uploads/{id}/chunks

Upload a single chunk (multipart form). Rate limit exempt for streaming.

FieldTypeRequiredDescription
chunkfileYesChunk data (max 50MB)
chunk_indexintegerYes0-based chunk index
{"upload_id": "uuid", "chunk_index": 0, "received": 1, "total_chunks": 7, "complete": false}

POST /api/v1/uploads/{id}/complete

Finalize upload: reassemble chunks and start transcription job.

{"success": true, "job_id": "uuid", "filename": "large_file.mp4", "file_size_mb": 300.0}

Health

GET /api/v1/health

Health check with queue stats, GPU info, and supported formats. Rate limit exempt.

{
  "status": "healthy",
  "gpu": {"available": true, "device": "NVIDIA GeForce RTX 4070"},
  "queue_size": 0, "processing": 0, "completed": 12,
  "supported_formats": ["aac","avi","flac","mp3","mp4","wav",...],
  "supported_models": ["base","large","medium","small","tiny"],
  "chunk_size_mb": 50, "max_request_mb": 55
}

Examples

Simple upload

curl -X POST https://transcript.fluximetry.com/api/v1/jobs \
  -F "files=@meeting.mp4" \
  -F "model=medium" \
  -F "language=en"

Check job status

curl https://transcript.fluximetry.com/api/v1/jobs/{job_id}

Download result

curl -O https://transcript.fluximetry.com/api/v1/jobs/{job_id}/files/meeting_transcription.txt

Chunked upload (large files)

# 1. Initiate
curl -X POST https://transcript.fluximetry.com/api/v1/uploads \
  -H "Content-Type: application/json" \
  -d '{"filename":"large.mp4","total_chunks":6,"model":"base"}'

# 2. Upload chunks (repeat for each)
curl -X POST https://transcript.fluximetry.com/api/v1/uploads/{upload_id}/chunks \
  -F "chunk=@chunk_0" -F "chunk_index=0"

# 3. Finalize
curl -X POST https://transcript.fluximetry.com/api/v1/uploads/{upload_id}/complete

Cancel a job

curl -X DELETE https://transcript.fluximetry.com/api/v1/jobs/{job_id}
